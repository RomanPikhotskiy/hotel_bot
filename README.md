# Telegram Hotel Bot

Учебный Telegram-бот для поиска отелей через RapidAPI (Booking) с сохранением истории запросов в SQLite.

Проект сделан на Python и хорошо подходит как портфолио для позиции **Automation QA Engineer (Python)**: есть модульная структура, работа с внешним API, БД, state-machine и автотесты на `pytest`.

## Что умеет бот

- Ищет отели по городу.
- Поддерживает команды:
  - `/lowprice` - самые дешевые отели.
  - `/highprice` - самые дорогие отели.
  - `/bestdeal` - отели ближе к центру.
  - `/history` - просмотр истории поиска.
- Запрашивает фильтры:
  - диапазон цены;
  - даты заезда/выезда;
  - количество взрослых и детей;
  - необходимость фото отелей;
  - количество отелей в выдаче.
- Сохраняет найденные результаты и историю в БД.

## Технологии

- Python 3.10+
- `pyTelegramBotAPI`
- `python-telegram-bot-calendar`
- `requests`
- `peewee`
- SQLite
- `pytest`

## Структура проекта

- `api/` - клиент для RapidAPI.
- `handlers/` - message/callback handlers бота.
- `states/` - состояния диалога.
- `database/` - модели и CRUD-операции.
- `keyboards/` - inline-клавиатуры.
- `utils/` - сервисные функции.
- `tests/` - автотесты на `pytest`.

## Используемые API

- [Booking.com API (RapidAPI)](https://rapidapi.com/DataCrawler/api/booking-com15)
  - `searchDestination`
  - `searchHotels`
  - `getHotelDetails`

## Быстрый старт

1. Клонировать репозиторий и перейти в папку проекта.
2. Создать и активировать виртуальное окружение.
3. Установить зависимости:

```bash
pip install -r requirements.txt
```

4. Создать файл `.env` на основе `.env.template` и заполнить:

```env
BOT_TOKEN=ваш_токен_бота
RAPID_API_KEY=ваш_rapidapi_ключ
```

5. Запустить бота:

```bash
python main.py
```

## Тесты

Запуск:

```bash
pytest -v
```

### Что покрыто тестами

- Валидация дат:
  - дата заезда не раньше текущей;
  - дата выезда не раньше даты заезда;
  - обработка невалидной даты.
- Обработка пользовательского ввода:
  - валидация числовых значений;
  - проверки границ (например, лимит количества отелей);
  - переходы между состояниями.
- История поиска:
  - хранение только последних 10 записей;
  - корректная отправка истории пользователю.
- API-клиент:
  - корректные endpoint'ы;
  - правильная сборка query-параметров;
  - форматирование дат и children-параметров.
- Работа с БД:
  - создание и сохранение сущностей;
  - защита от дублей пользователей;
  - сохранение найденных отелей.
